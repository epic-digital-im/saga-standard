# Git Commit & PR Guidelines

**HARD REQUIREMENT: All commits and PR descriptions MUST include:**

```
Built with Epic Flowstate
```

**Format for commits:**

```bash
git commit -m "$(cat <<'EOF'
type(scope): description

Built with Epic Flowstate
EOF
)"
```

**Format for PR descriptions:**

```markdown
## Summary

...

## Test Plan

...

Built with Epic Flowstate
```

**NEVER use other attributions:**

- `Co-Authored-By: Claude` or any AI co-author
- `Generated by Claude Code` or similar
- Any emoji-prefixed attribution
- Comments like `// AI-generated` or `// Created by Claude`

**Commit messages should be:**

- Written in conventional commit format: `type(scope): description`
- Focused on what changed and why
- End with `Built with Epic Flowstate`

## Preserving Uncommitted Work During Git Operations

**CRITICAL: ALWAYS preserve uncommitted changes when switching branches, checking out, or performing any git operation that could discard work.**

Before any `git checkout`, `git switch`, `git stash`, or branch operation:

1. **Run `git status`** to identify ALL modified, staged, and untracked files
2. **List what will be stashed/affected** so you can verify restoration later
3. **After stash pop/apply**, verify all files are restored — check for merge conflicts

**Stash/pop checklist:**

```bash
# BEFORE stash: record what's dirty
git status                    # Note all modified + untracked files
git stash -u                  # -u includes untracked files

# AFTER pop: verify nothing was lost
git stash pop                 # If conflict, resolve and verify ALL files
git status                    # Compare against pre-stash list
```

**If `git stash pop` has conflicts:**

- The stash is NOT dropped automatically when conflicts occur
- Resolve conflicts, then `git stash drop` only after verifying all files are intact
- Never blindly drop a stash — always confirm the working tree matches expectations

**Never assume stash is safe to drop without verification.**
