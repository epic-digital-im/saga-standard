Section 2: Data Architecture & State Management

How data flows through the app — storage, providers, and SDK integration

Data Flow Architecture
┌─────────────────────────────────────────────────────────────────────┐
│                          REACT TREE                                 │
│                                                                     │
│  <AppProvider>                                                      │
│    <StorageProvider>         ← Encrypted local DB (wallets, keys)   │
│      <AuthProvider>          ← Biometric/PIN unlock state           │
│        <ChainProvider>       ← viem clients, contract configs       │
│          <RelayProvider>     ← saga-client-rt WebSocket connection  │
│            <PaymentProvider>  ← x402 payment state & subscriptions  │
│              <Navigation/>                                          │
│            </PaymentProvider>                                       │
│          </RelayProvider>                                           │
│        </ChainProvider>                                             │
│      </AuthProvider>                                                │
│    </StorageProvider>                                               │
│  </AppProvider>                                                     │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│                       STATE MANAGEMENT                              │
│                                                                     │
│  Pattern: React Context + Custom Hooks (BlueWallet model)           │
│  No Redux/Zustand — contexts are sufficient for this domain         │
│                                                                     │
│  ┌─ StorageContext ──────────────────────────────────────────────┐  │
│  │                                                               │  │
│  │  wallets: Wallet[]          ← Self-custody + managed wallets  │  │
│  │  identities: Identity[]     ← Agent, Org, Directory NFTs     │  │
│  │  activeWalletId: string     ← Currently selected wallet       │  │
│  │  activeIdentityId: string   ← Currently active identity       │  │
│  │                                                               │  │
│  │  addWallet(w) / deleteWallet(id)                              │  │
│  │  addIdentity(i) / updateIdentity(id, patch)                  │  │
│  │  saveToDisk()               ← Debounced encrypted persist     │  │
│  │  loadFromDisk()             ← Decrypt & hydrate on unlock     │  │
│  │                                                               │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  ┌─ RelayContext ────────────────────────────────────────────────┐  │
│  │                                                               │  │
│  │  client: SagaClient         ← From saga-client-rt             │  │
│  │  connected: boolean                                           │  │
│  │  messages: Message[]        ← DMs (decrypted in memory only)  │  │
│  │  groups: Group[]            ← Group conversations              │  │
│  │  unreadCounts: Map          ← Per-conversation unread          │  │
│  │                                                               │  │
│  │  sendMessage(to, msg)                                         │  │
│  │  sendGroupMessage(groupId, msg)                               │  │
│  │  syncMemory(checkpoint)     ← Envelope recovery                │  │
│  │                                                               │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  ┌─ ChainContext ────────────────────────────────────────────────┐  │
│  │                                                               │  │
│  │  publicClient: PublicClient   ← viem read-only client         │  │
│  │  walletClient: WalletClient   ← viem signing client           │  │
│  │  chain: base | baseSepolia                                    │  │
│  │                                                               │  │
│  │  mintAgent(handle, hubUrl)                                    │  │
│  │  mintOrg(handle, name)                                        │  │
│  │  mintDirectory(directoryId)                                   │  │
│  │  registerHandle(handle, entityType, tokenId)                  │  │
│  │  resolveHandle(handle) → entity                               │  │
│  │  createTBA(tokenId, contractAddr)                             │  │
│  │                                                               │  │
│  └───────────────────────────────────────────────────────────────┘  │
│                                                                     │
│  ┌─ PaymentContext ──────────────────────────────────────────────┐  │
│  │                                                               │  │
│  │  subscriptions: Subscription[]                                │  │
│  │  paymentHistory: PaymentEvent[]                               │  │
│  │                                                               │  │
│  │  handleX402(response) → PaymentResult                         │  │

│  │                                                               │  │

Local Storage Architecture

Encrypted Storage Layers (following BlueWallet patterns)
┌─────────────────────────────────────────────────────────────┐
│                    SECURE KEYCHAIN                           │
│              (react-native-keychain)                         │
│                                                              │
│  • Master encryption key (AES-256)                           │
│  • Wallet private keys / mnemonics                           │
│  • saga-crypto KeyRing secrets                               │
│  • Biometric-protected access                                │
│  • WHEN_UNLOCKED_THIS_DEVICE_ONLY                            │
│                                                              │
└─────────────────────────────────────────────────────────────┘
                          │ encrypts
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                   ENCRYPTED REALM DB                         │
│              (realm with AES encryption)                     │
│                                                              │
│  Schemas:                                                    │
│  ┌─ WalletRecord ──────────────────────────────────────┐    │
│  │  id, type (self-custody|managed), label, address,    │    │
│  │  chain, encryptedSecret, balance, lastSync           │    │
│  └──────────────────────────────────────────────────────┘    │
│  ┌─ IdentityRecord ───────────────────────────────────┐     │
│  │  id, type (agent|org|directory), handle, tokenId,    │    │
│  │  contractAddress, tbaAddress, hubUrl, metadata       │    │
│  └──────────────────────────────────────────────────────┘    │
│  ┌─ MessageRecord ────────────────────────────────────┐     │
│  │  id, conversationId, from, to, scope, ciphertext,    │    │
│  │  timestamp, status (sent|delivered|read)              │    │
│  └──────────────────────────────────────────────────────┘    │
│  ┌─ DocumentRecord ───────────────────────────────────┐     │
│  │  id, agentHandle, exportType, sagaVersion,           │    │
│  │  encryptedContent, layers[], sizeBytes, checksum     │    │
│  └──────────────────────────────────────────────────────┘    │
│  ┌─ SubscriptionRecord ──────────────────────────────┐      │
│  │  id, tenantId, hubUrl, plan, status,                │     │
│  │  periodEnd, paymentHistory[]                        │     │
│  └──────────────────────────────────────────────────────┘    │
│  ┌─ GroupRecord ──────────────────────────────────────┐     │
│  │  id, name, members[], groupKeyId, lastActivity      │     │
│  └──────────────────────────────────────────────────────┘    │
│                                                              │

SDK Integration Model

How existing @epicdm packages map into the app
┌──────────────────────────────────────────────────────────────────┐
│                        APP FEATURES                              │
│                                                                  │
│  Messaging ──────────► @epicdm/saga-client-rt                    │
│  feature               • SagaClient (connect, sendMessage, etc.) │
│                         • RelayConnection (WebSocket)            │
│                         • MessageRouter (DM, group, memory)      │
│                         • PolicyEngine (governance)              │
│                                                                  │
│  Documents ──────────► @epicdm/saga-sdk                          │
│  feature               • assembleSagaDocument()                  │
│                         • validateSagaDocument()                 │
│                         • packSagaContainer() / extract()        │
│                         • encryptLayer() / decryptLayer()        │
│                                                                  │
│  Identity  ──────────► @epicdm/saga-client                       │
│  feature               • ChainClient (mint, register, resolve)   │
│                         • AuthClient (challenge-response)        │
│                         • ServerClient (REST API calls)          │
│                                                                  │
│  Wallet    ──────────► @epicdm/saga-crypto                       │
│  feature               • SagaKeyRing (key management)            │
│                         • seal() / open() (envelope encryption)  │
│                         • deriveX25519KeyPair()                  │
│                         • createEncryptedStore()                 │
│                                                                  │
│  Payments  ──────────► @x402/core + custom X402Client            │
│  feature               • Parse 402 responses                     │
│                         • Build payment proofs                   │
│                         • Sign with wallet (viem)                │
│                         • Retry with settlement proof            │
│                                                                  │
│  Directory ──────────► @epicdm/saga-client (ServerClient)        │
│  feature               • GET /v1/agents, /v1/orgs               │
│                         • GET /v1/resolve/:handle                │
│                         • On-chain reads via viem publicClient   │
│                                                                  │
│  On-Chain  ──────────► viem + contract ABIs from @epicdm/saga    │
│  ops                   • SAGAHandleRegistry ABI                  │
│                         • SAGAAgentIdentity ABI                  │
│                         • SAGAOrgIdentity ABI                    │
│                         • SAGADirectoryIdentity ABI              │
│                         • SAGATBAHelper ABI                      │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
