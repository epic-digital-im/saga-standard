Section 3: Wallet Management Model

Dual-track wallet system — self-custody and managed wallets as equal citizens

Wallet Class Hierarchy (inspired by BlueWallet's AbstractWallet)
                    ┌─────────────────────┐
                    │   AbstractWallet     │
                    │─────────────────────│
                    │ id: string           │
                    │ label: string        │
                    │ chain: Chain         │
                    │ address: string      │
                    │ balance: bigint      │
                    │ type: WalletType     │
                    │─────────────────────│
                    │ getBalance()         │
                    │ getAddress()         │
                    │ getTransactions()    │
                    │ signMessage(msg)     │
                    │ signTransaction(tx)  │
                    │ toJSON() / fromJSON()│
                    └─────────┬───────────┘
                              │
              ┌───────────────┼───────────────┐
              │                               │
  ┌───────────▼──────────┐      ┌─────────────▼────────────┐
  │  SelfCustodyWallet   │      │    ManagedWallet         │
  │──────────────────────│      │──────────────────────────│
  │ mnemonic: encrypted  │      │ managedId: string        │
  │ derivationPath: str  │      │ hubUrl: string           │
  │ passphrase?: enc     │      │ authToken: string        │
  │ hdNode: cached       │      │ provider: 'saga-hub'     │
  │──────────────────────│      │──────────────────────────│
  │ generateMnemonic()   │      │ signMessage(msg)         │
  │ importMnemonic(m)    │      │   → API call to hub      │
  │ exportMnemonic()     │      │ signTransaction(tx)      │
  │ deriveAddress(idx)   │      │   → API call to hub      │
  │ signMessage(msg)     │      │ requestExport()          │
  │   → local signing    │      │   → convert to           │
  │ signTransaction(tx)  │      │     self-custody         │
  │   → local signing    │      │                          │
  │ backup() → QR/file   │      │                          │
  └──────────────────────┘      └──────────────────────────┘

  User holds private keys          Hub holds keys on behalf
  Full sovereignty                 Lower friction onboarding
  Backup responsibility            Hub manages backup
  Signs locally                    Signs via authenticated API
  Works offline (signing)          Requires hub connection

Wallet Operations Matrix

What each wallet type can do
┌────────────────────────┬──────────────┬──────────────┐
│ Operation              │ Self-Custody │ Managed      │
├────────────────────────┼──────────────┼──────────────┤
│ View balance           │     ✓        │     ✓        │
│ View transactions      │     ✓        │     ✓        │
│ Send tokens            │  Local sign  │  Hub API     │
│ Receive tokens         │     ✓        │     ✓        │
│ Sign messages          │  Local sign  │  Hub API     │
│ Mint identity NFT      │  Local sign  │  Hub API     │
│ Register handle        │  Local sign  │  Hub API     │
│ x402 payments          │  Local sign  │  Hub API     │
│ Relay authentication   │  Local sign  │  Hub API     │
│ Export mnemonic        │     ✓        │  Request*    │
│ Backup to file/QR      │     ✓        │     ✗        │
│ Works offline          │  Signing ✓   │     ✗        │
│ Multiple addresses     │  HD derive   │  Hub-managed │
│ Cross-hub portability  │  Full        │  Via export  │
│ Key rotation           │  Manual      │  Hub-managed │
├────────────────────────┼──────────────┼──────────────┤
│ Upgrade path           │     —        │  → Self-     │
│                        │              │  Custody     │
└────────────────────────┴──────────────┴──────────────┘

* Export from managed → self-custody is a one-way
  sovereignty upgrade. Hub releases keys to user.

  Both wallet types implement the same AbstractWallet
  interface, so all app features work identically
  regardless of custody model.

Key Security Model

Encryption & Key Storage
┌────────────────────────────────────────────────────────────┐
│                    KEY HIERARCHY                            │
│                                                            │
│  Master Key (derived from user PIN + device key)           │
│    │                                                       │
│    ├── Wallet Keys                                         │
│    │   ├── Self-custody: BIP-39 mnemonic → HD derivation   │
│    │   │   └── Stored: AES-encrypted in Keychain           │
│    │   └── Managed: auth token for hub signing API         │
│    │       └── Stored: Keychain (refreshable)              │
│    │                                                       │
│    ├── SagaKeyRing (from saga-crypto)                      │
│    │   ├── Agent X25519 keypair (DM encryption)            │
│    │   ├── Group keys (per group)                          │
│    │   └── Org keys (company DERP encryption)              │
│    │   └── Stored: AES-encrypted in Realm                  │
│    │                                                       │
│    └── App Encryption Key                                  │
│        └── Encrypts Realm database at rest                 │
│                                                            │
│  Biometric Unlock:                                         │
│    Device biometric → unlocks master key → unlocks all     │
│    Fallback: PIN entry                                     │
│                                                            │
│  Managed Wallet Note:                                      │
│    SagaKeyRing is ALWAYS local, even with managed wallet.  │
│    Hub never sees encryption keys — only signing keys.     │
│    This preserves message privacy regardless of custody.   │
│                                                            │
└────────────────────────────────────────────────────────────┘
